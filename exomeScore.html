Notes on process to derive exome-wide risk score

This document sets out the process followed to derive an exome-wide risk score as described in this paper: http://biorxiv.org/content/early/2017/06/04/145961

This documentation should be read in conjunction with the paper, which provides background and further information.

The process involves a number of stages with custom-written software and specification files. The data set it was applied to is not publicly available (though can be obtained from dbGaP) so it is not possible to provide a full working example. However hopefully this description and the example files should allow people to apply it to their own datasets. A set of example files, referred to below, are provided in the directory called exampleFiles/exomeRiskScore. They will need to be modified before use.

To build the necessary programs described below, they first need to be added to the EXES variable in scoreassoc.mak. Then enter:

make -f scoreassoc.mak

Gene-wise extraction of variants with binary coded risk scores

The first step involves extracting genotypes for each variant from a vcf file containing exome sequence data for a sample of case-control subjects. The geneVarAssoc program is used for this purpose. It is available here:

https://github.com/davenomiddlenamecurtis/geneVarAssoc

Variants are extracted one gene at a time. The variants are annotated using VEP, SIFT and PolyPhen. Usually geneVarAssoc would allocate a weight to each variant depending on its functional effect. Here we wish to provide a binary encoded weight which says whether or not the variant has one of a number of attributes. To do this, a special argument file for geneVarAssoc was used called gvs.SSS2.vrare.1.arg. This assigns binary coded weights for VEP annotations based on those in allVarWeights.txt and then adds 64 to the weight if PolyPhen classes the variant as possibly or probably damaging and 128 if SIFT classes the variant as deleterious. It should be noted that the file is set up to only use variants with MAF<0.01. However if desired it could be modified to include common variants.

To run this extraction for one gene one would issue a command such as:

geneVarAssoc --arg-file gvs.SSS2.vrare.1.arg --gene DRD2

Aggregation of variant scores across variant attributes

The next step is to aggregate the scores for individual variants into scores for the different possible variant attributes, according to the attributes each gene possesses. This is achieved by a custom-written program called getVarScores. This can be built using the scoreAssoc distribution by adding getVarScores to the EXES variable in the makefile scoreassoc.mak. It runs on the file produced by geneVarAssoc containing the genotypes and binary coded weights of the variants:

geneName=DRD2

getVarScores --flagfile varFlags.txt --gcdatafile gva.$geneName.dat --locusfilterfile gva.$geneName.lf.par --locusweightfile gva.$geneName.lw.par --locusnamefile gva.$geneName.comm.par --filterfile gva.$geneName.filter.par --outfile gvs.$geneName.sao --scorefile gvs.$geneName.vsco --weightfactor 1.000000

It requires a file here called varFlags.txt which provides the binary codes for the attributes and which is provided among the example files. It produces a file for each gene called gvs.$geneName.vsco. The weighting factor of 1 means that no extra weight is given to rare variants, because only rare variants are used. If one wished to include common variants in the analysis one might change this to, for example, 10.

Script to run on all genes

A script which will run this extraction and variant aggregation process on each gene is provided as setupScoreScripts.sh. It runs geneVarAssoc and then getVarScores on every gene listed in a file called allGenes101115.txt. It produces scripts which are set up to be run as jobs on a cluster and the main script will need to be modified to run on the computer system used.

Combining gene-wise scores into a summary table

Once the above process has been completed one has a variant score file for each gene. This contains, for each subject, the total score for each attribute type. The next stage is to combine these gene-wise score files for each of a number of gene sets and write them into a single summary table. This is done using the combineGeneScores program, which can be built by adding combineGeneScores to the EXES variable in scoreassoc.mak. It is run with a command such as the following:

combineGeneScores combinedScores.txt allLists.txt 8 ../results/*.vsco

The file allLists.txt consists of the names of files, each of which contains a list of genes belonging to that gene set. The number 8 provides the number of attributes used which variants may possess.

Once this has been done one has a condensed dataset consisting of, for each subject, the aggregate scores for each attribute and gene set. This would be contained in the file combinedScores.txt.

Calculating t statistics and fitting weights for gene sets and attribute types

The fitScores program carries out a t test to compare a weighted sum of scores between cases and controls and can adjust the weights for gene sets and attribute types in order to maximise the t statistic.

In order to compile fitScores you need to first install the dlib library: http://dlib.net/

You may only need to download the dlib files without compiling any of them because the necessary files get included at the time that fitScores is compiled. You need to add fitScores to the EXES variable in scoreassoc.mak and you also need to set the DLIB variable to be the directory which contains the dlib directory.

The fitScores program is run by providing it with a number of arguments. Rather than being program options, these are commands which are executed sequentially. Each begins with a double hyphen and is followed by a single parameter. They can be provided on the command line or in an argument file. The possible arguments are listed below. A typical usage would be:

fitScores --arg-file doStepwise.arg

--arg-file

This argument is followed by the name of an file from which further arguments are read. Two argument files are provided as examples, doStepwise.arg and doAll.fifths.stepWise.arg.

--n-var-type

This is followed by a number giving the number of different variant attributes, in the example 8.

--n-gene-set

This is followed by a number giving the number of gene sets used, in the example 33.

--test-train-file

This is followed by the name of a file which has number of rows equal to the number of subjects. In each row are two entries which are either 0 or 1. The first entry denotes whether the subject is to be used in the training dataset and the second whether it is to be used in the test dataset. The example file testTrain.all.txt has two 1s in each row, indicating that every subject is to be used in both the training and test datasets. This file provides the information for the number of subjects in the dataset so this argument, and the two above, must be used before the ones below.

Other example files are named e.g. ttFifth.2.txt, which specifies that the second fifth of subjects should be used as the test dataset and the other four fifths for training. These files were produced using an excel spreadsheet in which the case-control status of each subject was listed. Then two additional columns were made which numbered the cases and controls sequentially. Then columns were made taking these numbers modulo 5 and then finally a pair of columns was used to select different fifths. The spreadsheet used to do this is provided as trainTest.xlsx.

--var-scores-file

This is followed by the name of the file containing the aggregated scores across variant attributes and gene sets, as produced by combineGeneScores.

--read-params-file

This is followed by the name of a file which contains a row for each gene set then attribute type. The first entry is the weight. The second entry is the name of the gene set or attribute. The third entry is 1 or 0, depending on whether the weight is fixed or allowed to change. An example is provided as paramsFree.txt.

--fit

If this is followed by 1 then the fitting procedure is used to change the non-fixed weights to find a set of values which maximises the t statistic in the training set.

--save-max-t

If this is followed by 1 then the value of t statistic achieved by any subsequent fitting procedure is saved. The t statistics obtained by other fitting procedures, notably when performing the stepwise procedure, can be compared with it and the difference used to determine whether an adequately close fit has been achieved. If it is followed by a 0 then new t values obtained from fitting procedures are not saved and the originally saved maximum value is left intact.

--test-train

This is followed by 0 or 1 to select whether the subsequent t test will be performed on the training or test dataset.

--ttest-file

This is followed by the name of a file to be written which will contain the results of a t test performed on the scores in the dataset specified by the --test-train argument.

--write-params-file

This is followed by the name of a file to which the fitted weights will be written. For each gene set and attribute type is output the weight, the name and then 0 or 1 to indicate whether it is free or fixed.

--do-grid

If this is followed by a 1 then subsequently whenever weights are output using --write-params-file additional values are appended to each row. The first of these is the t statistic achieved using the fitted weights. Next is output the low and high limits for the range that the weight can take without reducing the t statistic 1 less than the saved maximum. Next, the t statistics obtained by setting the weight to a range of values from -100 to 100 are output. This file can be input to excel and graphed to show how the t statistic is sensitive to the different models parameters. If it is followed by 0 then the additional values will not be output in subsequent uses of --write-params-file.

--stepwiseTol

This is followed by a number which sets the tolerance for carrying out the stepwise procedure, which is carried out. The training dataset is used. Parameters can be dropped if they do not reduce the t statistic from the saved maximum by more than this amount. At each stage the weight which can be set to zero with the least reduction is removed and then the remaining weights are re-fitted.

--write-scores-file

This is followed by the name of a file to which will be written the IDs of the subjects in the test dataset followed by the score for each subject using the current weights.

Example argument files for fitScores

The provided files were used to produce the results reported in the paper.

The file doStepwise.arg reads in the aggregated scores and starting parameters, fits the weights in the whole dataset, outputs the fitted weights with a grid of values then carries out the stepwise process to zero weights which do not impact much on the t statistic.

The file doAllFifths.stepWise.arg takes each fifth at a time as a test set and uses the other four fifths as a training set. The same process is repeated five times. The weights are fitted in the training set and the fitted weights are saved. A t test is carried out using the test set and the scores for the test set are saved. A t test is also performed using all subjects. The stepwise procedure is carried out using the training set and inessential weights are set to zero. The weights are saved. A t test is carried out on the test set. The scores for subjects in the test set are saved.

Analysis of output from fitScores

The example file analyse.fifths.r provides R commands to read in the scores obtained from the five fifths used as test datasets, concatenate them and perform t test and ROC analyses. The file makeTables.r makes the tables used in the paper.
